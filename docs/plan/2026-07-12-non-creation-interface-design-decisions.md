## 1. 文档目的

本文记录 2026-07-12 非创作功能设计访谈中已经确认的产品、信息架构、视觉、
交互和安全决策，作为账户中心、邀请返利、认证层和管理控制台原型及业务接入的
共同依据。

本文不重新决定首页、基础创作、私人图库和无限画布的具体交互。非创作页面复用
既有原型中的中性色、材质、控件和动效 Token，但不照搬隐藏导航、时间轮、点阵
画布、作品聚焦动效或选择后立即执行付费动作等创作场景交互。

现有 Dashboard 把普通用户功能和管理功能平铺在同一侧栏，无法继续作为目标结构。
本轮目标不是给旧 Dashboard 换皮，而是建立创作、账户和管理三个职责清楚的产品
空间，并关闭真实业务中的信息、权限和风险操作分歧。

## 2. 产品空间与总信息架构

### 2.1 三个产品空间

确认使用三个独立产品空间：

1. 创作空间：承载首页、基础创作、无限画布和私人图库。
2. 账户中心：承载套餐、积分、订单、用量、邀请返利、公告、工单和账户设置。
3. 管理控制台：只面向超级管理员，承载平台指标、用户、支付、返佣、公告、
   工单、后端池和系统配置。

三个空间共享视觉 Token 和明暗主题，但使用独立 URL、独立导航和符合各自任务的
布局。视觉统一不等于把三类任务塞进同一个 Dashboard 壳层。

### 2.2 空间切换

- 创作空间左下账户按钮打开账户菜单。
- 账户菜单提供“账户中心”和“管理控制台”入口。
- 管理入口只对 `super_admin` 显示。
- 账户中心只显示账户中心内部导航，并提供“返回创作”。
- 管理控制台只显示管理导航，并在导航底部提供“返回产品”和“账户中心”。
- 返回创作空间时恢复离开前的创作、图库或无限画布位置与草稿状态。
- 浏览器返回、前进、刷新和可分享 URL 必须保持空间与页面状态。
- 账户中心和管理控制台不继续套用创作空间的隐藏侧栏。

## 3. 账户中心

### 3.1 入口和默认页面

- 账户中心不设置独立概览首页。
- 进入账户中心后默认打开“套餐与积分”。
- 账户按钮弹层可以直接显示当前套餐、总积分和未读状态，但不替代账户页面。
- 删除旧控制台中与创作空间重复的创作数量、最近作品、快捷创作和价格图表。

### 3.2 内部导航

账户中心首版导航分为：

```text
资金与权益
├── 套餐与积分
├── 订单与用量
└── 邀请返利（按功能状态和参与历史显示）

服务
├── 公告
└── 支持工单

账户
├── 个人资料
├── 安全
└── 数据与账户
```

首版不提供“创作偏好”和“开发者能力”。现有外部 API、自定义上游 API、系统文档、
内容审核偏好和后端选择等底层能力暂时保留，但不进入新版账户导航与本轮原型。

### 3.3 套餐与积分

- 页面默认展示当前套餐、权益生效与到期时间、总积分和购买命令。
- 账户中心内可以直接更换套餐和购买积分，不要求登录用户返回首页 3D 定价场景。
- 公共定价场景继续服务未登录用户和公开套餐浏览。
- 两处复用同一套餐数据、能力矩阵、价格计算和购买组件，不维护两套价格逻辑。
- 页面只显示总积分，不显示积分批次、最近到期批次或到期预告。
- 积分到期后在积分流水中增加明确的“积分过期”记录。
- 不发送积分到期邮件。

### 3.4 固定期限套餐

所有付费套餐统一为固定期限权益；免费套餐没有权益到期时间：

- 暂不支持自动续费，不按支付渠道区分订阅行为。
- 公共定价和账户购买界面使用“1 个月、1 年”，不使用暗示自动续费的“月付、
  年付”。
- 用户侧展示生效时间、到期时间、续购或升级，不显示续费日期和取消订阅。
- 套餐有效期内只能升级到更高档套餐。
- 升级只允许选择与当前套餐相同的 1 个月或 1 年期限，不支持跨期限升级。
- 升级立即生效。剩余天数和未使用套餐积分分别折算可抵扣金额，实际抵扣取两者
  较小值；金额按当前报价币种的最小货币单位向下取整，最低实付 1 个最小货币单位。
- 原套餐剩余积分作废，新套餐发放完整周期积分，并从升级当天重新计算期限。
- 有效期内不能重复购买同档套餐，也不能提前购买低档套餐。
- 套餐到期后可以重新选择任意套餐；未购买新套餐时回到免费套餐。
- 不支持套餐叠加、预排续购和自动降档配置。

支付管理和用户页面中的“订阅记录”统一改名为“套餐权益记录”。现有支付渠道不支持
的取消按钮必须删除，不能保留点击后必然失败的命令。

切换固定期限模式前必须先处理现有自动续费契约：

- 新购买入口先停止创建自动续费订阅。
- 对仍可能自动扣款的存量订阅调用支付渠道的周期末取消，并核对渠道回执。
- 存量用户在当前已付周期结束前继续保留原权益。
- 渠道尚未确认停止续费时必须保留取消入口，不能先删除用户的止付能力。
- 渠道确认周期末取消后，该用户到期进入统一的固定期限套餐规则。

### 3.5 订单与用量

旧“账单与用量”和“使用记录”收敛为一个页面：

```text
积分流水（默认）
支付订单
使用记录
```

- 积分流水展示收入、消费、退款、过期、返利转入和余额变化。
- 支付订单展示积分包与套餐订单、支付状态、履约状态和真实订单参考号。
- 没有真实发票能力时不显示“发票”列，也不承诺下载发票。
- 使用记录只展示当前用户自己的请求，按时间、模型、来源、张数、状态和实际积分消耗展示。
- 支持按状态、模型和提示词筛选；筛选、页码和选中记录写入 URL，支持刷新、返回和前进。
- 点击记录打开只读详情，展示结果图、参考图、完整提示词、模型与尺寸、来源、实际积分和用户可理解的失败信息。
- 使用记录可以关联到图库作品；没有作品时仍保留财务与错误记录。
- 普通用户不看到内部渠道、调度成员、原始上游错误、其他用户或管理员字段。
- 桌面使用紧凑列表与右侧详情；手机使用记录列表与覆盖式详情，不依赖横向滚动。
- 积分增减和余额的账务真相来自 `credits_transaction`。
- 法币支付金额与支付状态来自支付订单和支付渠道回执，不能从积分流水反推。
- `generation` 只补充提示词、模型、状态和作品等生成上下文，不能作为扣费真相。
- 返利转入必须以正向积分收入显示，不能被误标为负数消费。
- 删除旧“使用记录”一级入口和永久兼容页面。

## 4. 邀请返利

### 4.1 产品定位

邀请返利页定位为可信的奖励账本，不是营销活动落地页。页面应让用户理解自己的
可用、冻结、已转换和已取消奖励，并能够追踪每次转换结果。

页面信息层级：

1. 邀请链接、邀请码、当前返佣比例和复制操作。
2. 可用奖励、冻结奖励、已转换奖励和累计奖励摘要。
3. 逐笔奖励流水，包含奖励积分、比例、状态、冻结或解冻时间和冲正结果。
4. 转积分记录与对应积分流水关联。
5. 固定奖励规则说明。

不使用营销式大标题、插画、卡片瀑布流或社交证明模块。

### 4.2 被邀请人隐私

- 用户侧完全不显示被邀请人的姓名、邮箱、脱敏邮箱、用户 ID 或匿名编号。
- 不显示被邀请人的订单号、支付渠道或精确消费金额。
- 奖励流水只展示奖励事实、比例、状态和时间。
- 退款或拒付只展示关联奖励已撤销及扣回结果。
- 管理控制台在超级管理员权限和审计约束下保留完整身份与订单信息。
- 用户侧 UOL 输出不得继续返回被邀请人的身份、消费额或订单字段，不能只在组件中
  隐藏现有 `ReferralOverview.invitees`。
- 用户概览、分页奖励账本和转换记录使用不含身份字段的专用输出；完整身份只允许由
  超级管理员 Operation 返回。

### 4.3 功能关闭策略

- 功能开启时正常显示账户中心入口，允许分享、绑定和转积分。
- 功能关闭但用户已有邀请关系或奖励时，保留入口与完整历史，页面标记计划已暂停。
- 暂停后不再产生新邀请，也不为既有邀请关系上的新支付增加返佣。
- 暂停前已经入账的冻结奖励继续按既定规则处理，可用奖励仍允许转积分。
- 功能关闭且用户从未参与时，完全隐藏入口，不展示空的“暂未开放”页面。
- 后台关闭功能前必须说明影响范围，不能让已有财务权益从界面中消失。

### 4.4 注册绑定

邀请绑定只发生在新账号注册流程：

- 移除归因 Cookie 及相关读取、写入和有效期配置。
- 邀请注册链接只依赖邀请码查询参数，沿用现有 `aff` 参数。
- 注册认证层显示可编辑的邀请码输入框，非必填。
- 链接参数用于预填，注册人可以替换或清空邀请码。
- 不增加额外的邀请关系说明或确认文案。
- 只有创建新账号成功时才能绑定邀请关系。
- 已有账号访问邀请链接时不绑定，也不修改现有关系。
- 不提供注册后 72 小时补录或账户中心手动绑定入口。
- 绑定成功后永久锁定，不允许用户或管理员解除或更换。
- 参数在注册前丢失时不追溯归因。

邀请码输入与失败规则：

- 空邀请码不影响正常注册。
- 已填写的邀请码必须在创建账号前校验格式、存在性和功能开关。
- 邀请码无效或功能已暂停时在输入框显示错误，用户清空或修正后才能继续。
- 用户保留邀请码提交时，账号创建与邀请绑定必须作为一个原子、幂等的注册结果。
- 绑定服务异常时本次注册失败且不留下已创建但未绑定的账号，重试不能重复绑定。

绑定必须进入注册统一操作链路，不能依赖注册后恰好访问旧 Dashboard 页面才执行。
绑定失败不得产生半绑定关系或重复关系。

### 4.5 奖励转换

- 用户手动触发转换，不自动转换。
- 一次转换全部当前可用奖励，不支持部分金额。
- 冻结奖励不参与本次转换。
- 确认层展示可用奖励、预计到账积分和转换后总积分。
- 提交期间禁止重复操作，服务端继续使用幂等键兜底。
- 转换完成后生成独立记录，并与积分流水双向关联。
- 不提供现金提现、银行卡或第三方钱包入口。

## 5. 公告与支持

### 5.1 公告定位

公告定位为系统通知，不是内容资讯流：

- 账户中心保留公告页面，账户按钮显示未读圆点或数量。
- 公告不进入创作空间一级导航。
- 普通公告只有用户打开具体内容后才标记已读。
- 进入公告列表不能自动把全部生效公告标记已读。
- 高严重级公告可以在首页、创作、图库和账户中心顶部显示紧凑横幅。
- 普通高严重级公告在用户确认后收起。
- 维护中断或安全事件可以设置为不可手动忽略，直到过期或管理员下线。
- 首版仍使用全站受众，不增加用户分群。

### 5.2 Markdown 公告

- 公告内容使用 Markdown 文档格式。
- 管理端提供源码编辑与实时预览。
- 支持标题、列表、表格、引用、代码和安全链接。
- 禁止原始 HTML、脚本、内联事件和任意样式。
- 链接协议、跳转目标和渲染结果必须经过服务端校验与清洗。
- 用户端和管理预览使用同一渲染规则，避免发布前后不一致。

### 5.3 用户工单

- 普通用户只查看和回复自己的工单。
- 用户只能回复未关闭工单。
- 管理员回复后产生站内未读状态，首版不发送用户邮件。
- 用户打开具体工单后才清除该工单的未读状态。
- 账户按钮可以汇总工单与公告未读状态，但不新增独立通知中心。

## 6. 账户设置与停用

### 6.1 首版设置范围

首版设置只保留：

- 个人资料：名称、头像、语言。
- 安全：修改密码、查看当前会话、撤销其他会话。
- 数据与账户：停用账户。

不设置含义模糊的“高级”页。首版不做数据导出、创作偏好或开发者配置。

### 6.2 停用账户

首版只提供停用，不执行数据删除或匿名化：

- 界面必须称为“停用账户”，不能声称永久删除账户或全部数据。
- 停用后立即禁止登录并撤销全部现有会话。
- 保留账户资料、作品、画布、工单和全部业务记录。
- 停用期间固定期限套餐和积分仍按原时间自然到期，恢复后不补偿停用时长。
- 停用账户继续占用原邮箱，不能用同一邮箱注册第二个账号。
- 操作前明确告知用户数据不会被删除。
- 超级管理员可以在核验后恢复已停用账户。
- 停用和恢复均写入审计记录。
- 停用与恢复必须使用独立、幂等的 UOL Operation，禁止复用现有会删除认证信息并
  匿名化用户的账户删除 Action。
- 真实数据删除、匿名化和导出留到后续独立设计。

## 7. 认证层

### 7.1 统一覆盖层

- 登录、注册、忘记密码和重置密码全部使用覆盖当前页面的认证层。
- 首版不使用独立视觉认证页面。
- 认证状态进入 URL，保证邀请链接、重置链接、刷新、返回和前进有效。
- 直接访问认证地址时使用安全的公开页面作为背景，并自动打开认证层。
- 认证层下方不得渲染用户无权读取的受保护页面数据。
- 用户主动触发的认证层可以关闭，并恢复原页面状态。
- 注册层包含已确认的可选邀请码输入框。
- 创作页触发登录时保留提示词、参考图和参数；登录成功后由服务端重新校验并继续动作。
- 停用账户登录时明确提示无法登录并引导联系支持，不伪装成密码错误。

### 7.2 认证组件边界

覆盖层的外壳、URL 状态和返回行为可以按入口变化，但邮箱、密码、邀请码、验证、
错误映射和提交逻辑必须复用同一组件与服务端操作，避免形成多套认证实现。

## 8. 管理控制台

### 8.1 权限范围

- 首版管理控制台只面向 `super_admin`。
- `observer_admin` 和普通 `admin` 暂不显示管理入口，也不能访问管理路由。
- 现有角色类型可以保留兼容，但不设计角色专属页面或只读变体。
- 所有管理路由和 Operation 必须在服务端再次校验 `super_admin`，不能只依赖导航隐藏。

### 8.2 常驻导航

管理控制台使用默认展开、允许手动折叠但不自动隐藏的左侧导航：

```text
总览
└── 指标与趋势

用户与服务
├── 用户管理
└── 工单管理

生成运营
└── 使用记录

资金与增长
├── 支付订单
└── 邀请返佣

内容运营
└── 公告管理

平台管理
├── 生图后端池
└── 系统与定价设置
```

生图后端池是独立管理模块，不再埋在系统设置的三级标签中。首版不增加全局审计
日志导航；各资源检查器只展示与当前资源有关的审计片段。

### 8.3 总览页

管理首页只展示指标与趋势，首版不建设待办队列。

全局时间范围：

- 默认最近 24 小时。
- 支持 24 小时、7 天、30 天和自定义范围。
- 支持与上一等长周期比较。

信息层级：

1. 顶部指标条：生成图片数、平台成功率、P95 耗时、支付金额、积分消耗和新增用户。
2. 生成趋势：请求量、完成量、失败分类和处理耗时。
3. 资金趋势：支付订单、履约状态、积分发放、消费、退款和过期。
4. 用户趋势：新增、活跃、付费用户和工单量。
5. 平台趋势：后端可用率、调度切换、限流和冷却情况。
6. 高频错误排行与失败样本。

页面使用连续数据带和图表，不把每个数字包装成独立卡片。高频错误主表按错误原因
聚合；展开行或右侧检查器展示具体用户、完整提示词、时间、模型、通道和原始错误。

### 8.4 表格与检查器

用户、使用记录、工单、支付订单、返佣记录和公告统一使用紧凑表格与右侧检查器：

- 点击整行打开检查器，保留列表、筛选条件和滚动位置。
- 当前对象写入 URL 查询参数，支持刷新、返回和分享调查链接。
- `Esc` 和浏览器返回关闭检查器。
- 行内只保留极少的高频命令。
- 修改、财务、权限和危险操作集中在检查器中。
- 系统设置和生图后端池等复杂工具继续使用完整页面。
- 首版不提供任何表格批量操作、跨页选择或批量操作栏。

### 8.5 风险操作规范

首版不提供“直接执行后短时撤销”的可逆开关模式。所有管理写操作按以下规则执行：

- 启用、停用、封禁等状态变化展示影响，要求填写原因并确认。
- 财务、套餐、权限和返佣操作展示目标、变更前后值、金额和连带影响。
- 永久删除、密钥和认证配置要求输入目标标识或指定确认文本。
- 部分成功必须进入结果面板，逐项列出成功、失败和待恢复记录。
- 每次完成后展示审计编号和幂等结果。
- 重复请求明确说明未重复执行，不能伪装成一次新的成功。
- 密码、密钥、令牌和完整敏感请求体不得进入日志或回执。

首版不建设全局审计日志页面，但审计记录本身仍是高风险操作的强制要求。

## 9. 管理模块

### 9.1 用户管理

用户列表用于筛选和定位，检查器包含：

```text
概览：身份、状态、套餐、总积分、注册和最近活动
积分：批次、流水、发放和扣减
订单：支付订单和套餐权益
生成：最近请求、提示词、错误和实际扣费
API：Key 状态、配额和最近使用，不回显密钥
支持：用户工单与回复记录
审计：与该用户有关的管理员操作
```

检查器操作菜单保留编辑资料、修改套餐、积分调整、封禁、冻结、恢复停用账户、
撤销会话和重设密码。

超级管理员可以直接设置用户新密码：

- 永远不能查看用户原密码。
- 新密码是正式密码，不标记为临时密码。
- 重设成功后撤销该用户全部现有会话。
- 用户下次登录后不强制修改密码，由用户自行决定。
- 密码内容不得写入日志、审计记录或成功回执。
- 首版不发送激活链接或密码重置邮件。

### 9.2 使用记录

使用记录是超级管理员调查全站生图请求的独立管理页，不与普通用户的
“订单与用量”复用同一个角色变形视图，也不只依附于单个用户检查器。

- 默认按创建时间倒序，每页 20 条。
- 支持按用户、状态、模型、提示词、开始日期和结束日期筛选。
- 表格稳定展示图片、用户、渠道、提示词、模型、尺寸、实际积分、状态、耗时和日期。
- 检查器展示生成 ID、请求 ID、完整提示词、提示词修复说明、来源、
  实际渠道、结果图、参考图、积分流水引用和原始错误。
- 筛选、页码和当前记录写入 URL，支持刷新、返回、前进和分享调查链接。
- 页面严格只读，不提供删除、重试、调整积分或批量操作。
- 生成请求和图像上下文来自生成记录；实际扣费以
  `credits_transaction` 双重记账为真相，不从历史 `generation` 行反推。

### 9.3 工单管理

- 从普通用户支持页拆出独立管理页面。
- 默认展示待处理和处理中工单。
- 支持按状态、未读、用户和关键词筛选。
- 使用表格与右侧会话检查器。
- 检查器同时展示用户套餐、总积分、关联订单和关联生成记录。
- 回复、修改状态和关闭工单在检查器中完成。
- 首版不增加负责人分配、SLA、内部备注、附件或批量操作。
- 用户支持路由与管理工单路由不复用同一套角色变形视图。

### 9.4 支付管理

支付管理首版严格只读：

```text
支付订单（默认）
到账账本
套餐权益记录
```

- 支付状态与积分履约状态分列展示。
- 点击订单打开检查器，串联用户、支付回调、积分批次和积分流水。
- 到账真相来自 `credits_batch` 和 `credits_transaction`。
- 可以复制订单号、打开用户详情和查看上下文审计。
- 不允许手工修改订单状态、标记到账、退款或补写套餐权益。

### 9.5 返佣管理

返佣管理默认进入“奖励账本”：

```text
奖励账本（默认）
邀请关系
邀请档案
转换记录
```

- 奖励账本展示订单来源、邀请双方、计佣金额、比例、积分、冻结时间和状态。
- 邀请关系展示永久绑定记录。
- 邀请档案展示邀请码、专属比例和累计数据。
- 转换记录展示积分批次和幂等结果。
- 全局搜索覆盖邮箱、用户 ID、邀请码和订单号。
- 运营界面使用百分比输入返佣比例，不要求直接填写 `bps`。
- 取消返佣前预览受影响账本、预计扣回积分和余额不足风险。
- 部分冲正失败时展示逐项结果，并保留继续调查入口。

### 9.6 公告管理

- 使用表格和检查器管理草稿、定时发布、过期、置顶、优先级和严重级别。
- 编辑器使用统一的安全 Markdown 源码与预览。
- 上下线和删除遵循风险操作规范，不提供批量操作。
- 首版只支持全站受众。

### 9.7 生图后端池

生图后端池独立为完整工具页面：

- 默认进入“API 后端”。
- 一级视图为分组、账号池、API 后端、Adobe 后端和接入工具。
- “同步 Sub2API”和“注册机”合并到低频的接入工具视图，再使用内部标签区分。
- 分组、账号和后端使用紧凑表格与检查器完成新增、编辑和状态查看。
- 新增与编辑表单不长期占据页面左侧。
- 健康、额度、并发、冷却和最近错误是稳定、可排序的状态列。
- 只有 API 后端行提供真实生图测活；测活直接执行，不弹二次确认。
- API 测活会提交 `n=1` 的真实上游生图任务，但不扣站内用户积分。上游一旦受理，
  可能计入供应商图像额度或账单；鉴权或连通失败未必产生费用，超时或手动停止也
  不能保证上游没有继续执行或计费。
- 测活按钮附近明确展示上述成本边界，并显示运行、停止和结果状态。
- 首版不暴露成员复选框、批量启停、批量分组、批量并发调整、批量刷新、批量恢复、
  错误账号批量清理或批量删除等表格批量操作界面。
- 已有批量 service、UOL 定义和并发可靠性实现继续保留，不因隐藏界面而删除。
- 接入与后台维护任务不属于表格批量编辑，继续保留，包括 Sub2API 同步、RT/AT
  导入、Adobe Cookie 导入、注册机批次和自动号池维持。

### 9.8 系统与定价设置

- 左侧使用可搜索的设置分类目录，右侧只显示当前分类。
- 每个分类独立保存，不再整页统一提交。
- 固定变更栏展示未保存字段和修改前后差异。
- 标记每个值来自数据库、环境变量还是系统默认值。
- 标记保存后立即生效、需要重启或需要重新构建。
- 环境变量覆盖的字段只读，不能产生“保存成功但不生效”的假象。
- 密钥只支持替换，不回显原值。
- 保存要求填写原因；敏感配置使用高风险操作流程。
- 使用配置版本检测并发修改，发生冲突时拒绝静默覆盖。

## 10. 视觉与主题

### 10.1 共用 Token

账户中心和管理控制台复用既有原型的：

- 中性黑白灰颜色体系。
- 近不透明哑光表面和弱亮边。
- 4px 至 8px 圆角，默认 6px。
- 窄而深的阴影。
- 紧凑图标按钮、菜单、分段控件、输入控件和工具提示。
- 等宽数字特性和稳定控件尺寸。

不复用首页 3D 场景、作品聚焦转场、点阵画布、隐藏时间轮或边缘自动展开导航。

### 10.2 全局主题

- 基础创作、私人图库、无限画布、账户中心和管理控制台共用同一明暗主题偏好。
- 主题选择保存在当前浏览器，不跨账号同步。
- 管理控制台不强制浅色或深色。
- 两种主题下保持相同的信息层级、表格密度、状态含义和可访问性。
- 首页是否跟随同一主题不由本文改写，继续以创作决策稿和后续原型校准为准。

### 10.3 受控语义色

- 背景、面板、表格和主要按钮继续使用中性色。
- 成功允许低饱和绿色，警告允许琥珀色，错误与危险操作使用红色，信息状态允许
  低饱和蓝色。
- 语义色只用于图标、状态点、文字和细边，不使用大面积彩色背景。
- 图表使用固定、低饱和的数据色，不使用渐变、霓虹或装饰色。
- 颜色不是唯一识别方式，必须同时提供文字、图标、线型或数值。

### 10.4 管理信息密度

- 管理页面以扫描、比较和重复操作效率为优先。
- 表格行高、筛选栏和分页尺寸保持稳定，加载状态不能改变列宽或布局。
- 页面区块使用连续数据带或无框布局，不把每个指标包装成卡片。
- 表格内不堆叠大段说明；复杂说明进入检查器或帮助文本。
- 检查器宽度使用稳定的响应式约束，不因内容动态改变。

## 11. 响应式范围

### 11.1 账户中心

- 完整支持手机、平板和桌面。
- 手机端使用顶部标题和抽屉导航，不压缩为难以点击的图标栏。
- 套餐购买、邀请链接、积分流水、公告、工单和账户停用均可在手机完成。
- 长表格在手机端改为分组记录列表，不依赖横向滚动。
- 文字、金额、状态和操作不得重叠或溢出。

### 11.2 管理控制台

- 首版只支持宽度不低于 1024px 的桌面环境。
- 小于 1024px 时显示明确的“请使用桌面端”页面。
- 小屏不渲染被压缩或容易误触的管理写操作。
- 不额外制作管理首页移动只读版。

## 12. 高保真原型范围

原型继续使用真实 React、CSS 和本地模拟数据，不连接支付、积分、密码、后端测活、
系统设置或其他真实写操作。

### 12.1 账户原型

1. 套餐与积分：免费套餐、付费套餐、购买套餐和购买积分。
2. 订单与用量：积分流水、支付订单、使用记录、请求详情及手机记录列表。
3. 邀请返利：正常账本、无记录、功能暂停、转换确认和转换结果。

账户原型至少检查桌面和手机视口。

### 12.2 管理原型

1. 管理总览：24 小时指标、趋势图、高频错误和失败请求检查器。
2. 用户管理：筛选表格、用户检查器、积分调整和密码重设。
3. 使用记录：全站筛选表格、URL 分页与只读请求检查器。
4. API 后端池：列表、检查器、测活状态和接入工具入口。
5. 系统设置：分类搜索、未保存差异、环境变量覆盖和保存冲突。

管理原型只验证 1024px 及以上视口。

### 12.3 原型安全边界

- 所有原型命令只改变本地模拟状态。
- 不调用真实 Server Action、UOL Operation、支付 SDK、邮件、上传、数据库或外部 API。
- 密码、密钥和用户身份使用明确的虚构数据。
- 测活、积分调整、返佣转换和设置保存必须有清楚的“模拟”状态，不得产生真实副作用。

## 13. 工程与数据约束

- 新增或改造的业务能力先在 UOL Operation Registry 中定义，再由页面薄适配调用。
- 账户页面不能绕过 `protectedAction` 或 Operation 权限网关。
- 管理传输层统一使用 `superAdminAction`，对应 UOL Operation 也必须声明超级管理员
  权限；客户端隐藏不能代替服务端权限检查。
- 财务展示、转换、发放、扣减、退款和冲正继续以双重记账和现有幂等键为真相。
- 奖励转换、积分调整、返佣冲正、密码重设、账户停用和恢复均需幂等或防重放保护。
- 套餐能力继续以 `plan-capabilities.ts` 为唯一来源。
- 用户侧资源访问继续校验归属，管理侧资源访问继续写审计记录。
- Markdown 公告必须使用结构化解析和安全渲染，禁止用字符串替换拼接 HTML。
- 表格筛选、检查器选择和标签状态进入 URL，服务端继续校验所有查询输入。
- 可选日志、邮件和监控服务未配置时优雅降级，不吞业务错误。

## 14. 验收标准

### 14.1 信息架构

- 创作、账户和管理三个空间在导航与 URL 上边界明确。
- 账户中心无低价值概览页，默认进入套餐与积分。
- 普通用户导航不再混入管理入口。
- 生图后端池不再埋在系统设置内部。
- 暂缓功能不出现在新版导航中。

### 14.2 用户可信度

- 套餐明确为固定期限权益，不出现自动续费或取消订阅误导。
- 总积分和每笔积分流水的正负方向正确。
- 积分过期产生可理解的流水记录。
- 返利账本不暴露被邀请人身份或消费信息。
- 功能暂停不会隐藏已有返利权益。
- 停用账户明确说明保留数据，不声称已删除。

### 14.3 管理效率与安全

- 所有列表保持筛选、分页、滚动和检查器 URL 状态。
- 管理首页在默认 24 小时范围内清楚呈现指标和趋势。
- 高频错误可以追踪到具体用户、提示词和请求上下文。
- 所有写操作展示目标、影响和结果，危险操作不能只以 toast 收尾。
- 支付管理不能改写财务事实。
- 密码和密钥从不回显或进入日志。
- 管理页面在小于 1024px 时不暴露可误触写操作。

### 14.4 可访问性与响应式

- 账户中心在手机、平板和桌面均可完整操作。
- 表格手机形态不依赖横向滚动。
- 检查器、认证层、确认层和 Markdown 内容具备正确焦点顺序与键盘关闭行为。
- 状态不只依赖颜色，语义色在深浅主题下均满足对比度要求。
- 减少动态效果时关闭非必要位移，只保留短淡入淡出。

## 15. 明确延期项

以下功能不属于首版，不应由实现者自行补回：

- `observer_admin` 和普通 `admin` 的管理页面或只读变体。
- 管理首页待办队列。
- 管理表格批量操作界面；已有底层批量能力继续保留。
- 全局审计日志查询页面。
- 管理控制台移动端或小屏只读版。
- 工单负责人、SLA、内部备注、附件和用户邮件通知。
- 公告用户分群、富文本 HTML 和图片编辑器。
- 账户创作偏好和开发者能力。
- 独立视觉认证页面。
- 自动续费、套餐叠加和预排续购。
- 积分批次列表和到期预告。
- 返利部分转换和现金提现。
- 注册后补录邀请码、归因 Cookie 和邀请关系解除。
- 账户数据删除、匿名化和导出。
- 全局通知中心。

这些延期项若后续启用，必须重新经过产品、权限、数据和视觉决策，不能直接恢复旧页面。
